# AI 竞赛学习助手 - 技术说明文档

## 一、项目概述

### 1.1 项目简介
AI 竞赛学习助手是一个基于人工智能的智能竞赛信息管理平台，帮助参赛者和指导教师快速获取竞赛信息、智能提取关键内容、一键跳转报名入口。

### 1.2 核心功能
- **智能角色跳转**：根据用户身份（参赛者/教师）自动跳转至对应的竞赛官网入口
- **多格式解析**：支持 PDF、网页（HTML）、TXT 等多种格式的竞赛文档智能解析
- **AI 信息提取**：自动抽取报名截止日期、参赛要求、联系方式等关键信息
- **双角色服务**：参赛者快速报名，指导老师管理学生

### 1.3 技术栈

| 层级 | 技术选型 |
|------|----------|
| 后端框架 | FastAPI (Python 3.10+) |
| 数据库 | SQLite (aiosqlite 异步驱动) |
| 前端框架 | Vue 3 + Composition API |
| UI 组件库 | Element Plus |
| 状态管理 | Pinia |
| HTTP 客户端 | Axios |
| 认证方式 | JWT (JSON Web Token) |
| AI 集成 | 多提供商架构 (Google Gemini, Cloudflare, Cerebras) |

|------|------|----------|

### 5.2 数据规范化处理

#### 5.2.1 URL 规范化
- 自动补全协议（优先 https）
- 验证 URL 格式有效性
- 三入口兜底：缺失时使用 source_url 填充

#### 5.2.2 日期规范化
支持格式转换：
- `YYYY/MM/DD` → `YYYY-MM-DD`
- `YYYY.MM.DD` → `YYYY-MM-DD`
- `YYYY年MM月DD日` → `YYYY-MM-DD`
- `YYYY年MM月` → 补充为月末

#### 5.2.3 联系方式规范化
- 按分隔符分割（`;` `；` `,` `，` `\n`）
- 识别联系人+电话模式
- 提取邮箱、QQ、微信等

### 5.3 置信度评估
- 有效 URL 格式检查
- 日期格式验证
- 非空数组检查
- 联系方式有效性（邮箱/手机号）
- notes 不确定性扣分

### 5.4 竞赛重名检测算法

保存竞赛时会检测是否存在相似竞赛，检测规则按优先级：

| 优先级 | 规则 | 示例 |
|--------|------|------|
| 1 | 完全相同 | A == B |
| 2 | 去除标点后相同 | "蓝桥杯——项目赛" ≈ "蓝桥杯项目赛" |
| 3 | 包含关系 | "蓝桥杯" ⊂ "蓝桥杯大赛" |
| 4 | 去除年份/届数后相同 | "第十六届蓝桥杯" ≈ "第十五届蓝桥杯" |
| 5 | Jaccard 字符相似度 > 80% | 字符集重叠度高 |
| 6 | 去除前后缀后相同 | "全国蓝桥杯竞赛" ≈ "中国蓝桥杯大赛" |

移除的标点符号：`——`、`-`、空格、`·`、`、`、`，`、`；`、`：`、括号、引号等

## 八、核心业务流程

### 8.1 用户注册登录流程
```
用户填写表单 → 前端验证 → 调用 API → 后端验证 → 
密码哈希/验证 → 生成 JWT → 返回 Token → 存储 localStorage
```

### 8.2 AI 提取流程
```
选择提供商 → 选择来源类型 → 输入内容 → 
调用提取 API → 内容获取（URL抓取/文件解析/直接使用） → 
调用 AI 服务 → 解析 JSON 响应 → 数据规范化 → 
置信度计算 → 保存记录 → 返回结果
```

### 8.3 竞赛保存流程
```
AI 提取结果 → 打开保存对话框 → 填写/编辑信息 → 
检查重名 → [无重名] 直接保存
           → [有重名] 选择处理方式：
              - 智能合并 → AI 合并 → 预览确认 → 保存
              - 覆盖更新 → 直接更新
              - 另建新条目 → 创建新记录
```

### 8.4 角色跳转流程
```
用户点击跳转 → 获取用户角色 → 查询竞赛信息 → 
根据角色选择 URL（entrant_url/teacher_url/default_url） → 
返回跳转链接 → 前端打开新窗口
```
